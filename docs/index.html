How it works

Three steps, about a minute.

Install the module

One composer require and a setup:upgrade. Works on any Magento 2.4+ store.

Mint a token

Create a bearer token for an admin user from the CLI. Tokens inherit that admin's role, so the AI can only do what that role can already do.

Connect your AI

Paste the URL and token into Claude, ChatGPT, Cursor, or any MCP-aware tool. It discovers what's available and goes to work.

Built for trust

The safe way to hand the AI a key to your store.

Bearer tokens, admin-role ACL inheritance, write gating, full audit trail. Every safety control you'd build by hand — built in.

Bearer tokens, hashed at rest

Plaintext is shown once. The database stores a bcrypt hash. A revoked token fails auth immediately.

Inherits admin permissions

Every call re-checks the owning admin's role. What a role can't see in admin, the AI can't see either.

Writes off by default

Write operations require two switches: a global store-config flag and a per-token flag. Either one off → blocked.

Full audit trail, PII redacted

Every call is logged with method, arguments, duration, admin, and token. Sensitive fields are fingerprinted before they hit disk.

One-click kill-switch

Flip a single store-config toggle and every request returns HTTP 503 before auth even runs.

Origin allowlist

Only browsers and tools from an admin-controlled list can even reach the endpoint — a built-in defense against DNS rebinding.

For developers

Add your own tools. Keep every safety rail.

Ship a new tool in a satellite module and inherit transport, authentication, ACL, audit, and PII redaction — for free.

  • One interface to implement, one DI line to register.
  • Every tool gets bearer auth, ACL checks, and audit rows automatically.
  • PHPStan level 9, strict types, full unit test scaffolding.
Read the developer guide 
// Register a new tool — that's it.
<type name="Magebit\Mcp\Model\Tool\ToolRegistry">
  <arguments>
    <argument name="tools" xsi:type="array">
      <item
        name="catalog.product.get"
        xsi:type="object">
          Vendor\Module\Tool\CatalogProductGet
      </item>
    </argument>
  </arguments>
</type>
Install

Up and running in 60 seconds.

$ composer require magebitcom/module2-mcp-module

After install, run bin/magento module:enable Magebit_Mcp, then setup:upgrade. The README has per-client setup snippets for Claude, ChatGPT, Cursor, and more.
