Build on the Magebit MCP core.

The core module ships the transport, authentication, authorization, audit, and tool-registry primitives — satellite modules add domain-specific tools that inherit all of them. This page is the developer's map.

What the core module ships

Install magebitcom/module2-mcp-module and you get:

  • JSON-RPC 2.0 over HTTP at a single POST /mcp endpoint. MCP protocol version 2025-06-18.
  • Bearer-token auth: plaintext shown once, bcrypt-hashed at rest.
  • Per-tool ACL enforcement — every tools/call re-checks the tool's ACL resource, and tools that implement UnderlyingAclAwareInterface also re-check the Magento admin-UI resource they delegate to.
  • Write-tool gating via a global store-config kill-switch and a per-token allow_writes flag.
  • Origin allowlist for DNS-rebinding defense.
  • Audit log of every call with PII fingerprinted to HMACs before write.
  • Admin UI for tokens and audit, plus store-config for name/description/kill-switch/origins/retention.
  • Console commands for token lifecycle, tool listing, and CI-ready ACL validation.

Request pipeline

Every POST /mcp runs this fixed sequence. Every stage writes an audit row from the finally block — unauthenticated attempts leave a trail.

  1. Origin header — DNS-rebinding defense; allowlist in store config.
  2. Bearer auth — 401 + WWW-Authenticate: Bearer on failure.
  3. Body parse — JSON-RPC envelope; body capped at 256 KiB so unauthenticated attackers can't OOM FPM pre-auth.
  4. Protocol version — Mcp-Protocol-Version header (required on every request after initialize).
  5. Dispatch — routes by method to the DI-registered handler, passing an authenticated context.
  6. Audit — writes the row; sensitive fields are fingerprinted by HMAC keyed on crypt/key.

Core contracts

Satellite modules reuse these five contracts — never duplicate them.

ContractRole
ToolInterfaceEvery MCP tool implements this. Registered by DI array into ToolRegistry.
UnderlyingAclAwareInterfaceOpt-in second ACL check. Returns the underlying admin-UI resource so MCP can't do what the admin UI can't.
FieldResolverInterfaceMarker for the field-resolver pattern used by satellite read tools.
ToolRegistryInterfaceRegistry contract.
ToolResultInterfaceResult envelope returned from every tool call.

Cross-cutting concerns (rate limiting, custom audit sinks, result masking) plug in via the magebit_mcp_tool_call_before and magebit_mcp_tool_call_after events.

Registering a tool

Implement Magebit\Mcp\Api\ToolInterface, declare an ACL resource under your module's acl.xml, and add a line to etc/di.xml:

<type name="Magebit\Mcp\Model\Tool\ToolRegistry">
  <arguments>
    <argument name="tools" xsi:type="array">
      <item name="catalog.product.get" xsi:type="object">
        Vendor\Module\Tool\CatalogProductGet
      </item>
    </argument>
  </arguments>
</type>

The registry validates at construction that the di.xml key matches getName() and matches the pattern ^[a-z][a-z0-9_]*(\.[a-z][a-z0-9_]*)+$. Duplicates fail setup:di:compile.

ACL convention

Tool ACL resources live under Magebit_Mcp::tools:

  • Tool name catalog.product.get → ACL resource Vendor_Module::mcp_tool_catalog_product_get. Dots map to underscores because ACL ids can't contain dots.
  • Group under a top-level Vendor_Module::mcp node nested under Magento_Backend::system, never under a wide-allow resource.

Four admin-UI resources gate the module itself: Magebit_Mcp::mcp_tokens, Magebit_Mcp::mcp_audit, Magebit_Mcp::config, Magebit_Mcp::tools — intentionally separate, so a token-manager role need not see the audit log and vice versa.

Console commands

CommandPurpose
magebit:mcp:tools:listEvery registered tool, its ACL resource, its write mode, whether it requests confirmation.
magebit:mcp:tools:validate-aclFail if any tool references an ACL resource not declared in acl.xml. Wire into CI.
magebit:mcp:token:createMint a bearer token for an admin user. Plaintext is shown once.
magebit:mcp:token:listList tokens (last used, revoked state, owning admin).
magebit:mcp:token:revokeRevoke a token — auth fails immediately, audit rows are preserved.
magebit:mcp:token:deleteHard-delete a token and keep audit rows pointing to deleted-<id>.
